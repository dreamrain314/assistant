二、AI 意图识别（classify_and_extract）
--------------------------------------------------------------------------------
  一次 AI 调用同时完成意图判断 + 数据提取，支持 6 种意图：

  add       录入    "张三有一个XX任务，后天要交"
                     提取 name/action/status/deadline → 入库
  query     查询    "今天有哪些任务"、"XX在哪天"、"我还有什么任务"
                     生成SQL → 注入team_id → 执行 → 口语化 + 模糊匹配
  delete    删除    "清空今天的任务"、"把这个删掉"
                     预览 → 反问确认 → 确认后执行
  update    更新    "把XX改成已完成"、"XX改到后天"
                     更新 status 和/或 deadline
  confirm   确认    "确定"、"是的" → 执行暂存的 pending 操作
  cancel    取消    "算了"、"取消" → 丢弃暂存操作

  智能特性：
  - 多任务检测：一句话含多个任务时，自动截取第一个并提醒"一次一个"
  - 多人检测：识别"张三和李四写报告" → 自动为每人各创建一条任务
  - 代词清洗：action 中自动将"我/你"替换为实际姓名，"他/她"替换为"其"
  - 去重确认：布置任务时自动检测相似已有任务，自然反问是否为同一个


三、时间理解能力
--------------------------------------------------------------------------------
  今晚十点前          当天 22:00
  今天要交/今天完成    当天 23:59
  明天下午3点         明天 15:00
  后天/后天上午       +2天
  大后天              +3天
  本周五前            本周五 18:00
  下周一              下周一 09:00
  本周六/本周日        对应日期 18:00
  7月份工作开题报告    "7月份"=主题描述，不误判为 deadline

  validate_deadline() 自动校验并规范化日期格式
  温度=0，同一输入永远返回相同结果


四、用户与权限系统
--------------------------------------------------------------------------------
  【小组机制】
  - 支持多小组数据隔离（team_id 强制过滤）
  - 创建小组时自动创建管理员账户
  - 管理员可注销小组（删除所有数据，需密码验证）
  - 登录两步：先选小组 → 再选成员/输入姓名

  【用户角色】
  - 管理员 (is_admin)：查看全部任务、管理成员、修改权限、注销小组
  - 普通成员：
    - can_create：可布置任务（默认开启）
    - can_remind：可催促他人（默认关闭）
  - 权限由管理员在"用户管理"面板中设置

  【账户安全】
  - 密码：werkzeug hash 存储，默认 123456
  - 修改密码：需旧密码验证
  - 密保问题：首次登录强制设置，用于找回密码
  - 忘记密码：输入姓名 → 显示密保问题 → 回答正确 → 重置为 123456
  - 管理员可重置他人密码

  【任务可见性】
  - 🌐 公共任务：小组内所有人可见
  - 👥 指定人员：仅指定成员可见
  - 非管理员查他人任务 → 只显示公开任务
  - 成员列表从 /api/teams 获取（无需管理员权限）


五、任务看板（右侧面板）
--------------------------------------------------------------------------------
  【卡片视图】
  - 每个任务一张卡片：负责人、任务描述、状态标签、截止时间
  - 超时任务：红色边框 + 🚨超时标签 + 已超时N小时
  - 已完成任务：绿色左边框 + 半透明
  - 可见性标签：🌐公共 / 👥仅限

  【排序与筛选】
  - 智能排序（默认）：超时→未完成→已完成，同组内按 deadline 升序
  - 时间排序：按创建时间倒序
  - 全部/我的任务切换
  - 搜索框：实时搜索姓名/任务/日期

  【卡片操作】
  - ✅ 完成按钮：标记任务为已完成
  - 📣 催促按钮（需 can_remind 权限）：发送通知给负责人
  - ✕ 删除按钮（管理员或有布置权限者可见）
  - 5秒冷却防重复催促

  【快速布置弹窗】
  - ➕ 按钮打开表单：任务内容、负责人、截止时间(datetime-local)、可见性


六、通知与催促系统
--------------------------------------------------------------------------------
  - 🔔 通知铃铛：显示未读数量徽章
  - 下拉列表：查看通知详情 + 单条标记已读 + 全部已读
  - 每 20 秒自动轮询新通知
  - 新通知弹 toast 提示
  - 催促消息格式："XX 提醒你尽快完成：任务描述（已超时）"
  - 通知存储到 notifications 表 (Supabase)


七、对话与交互
--------------------------------------------------------------------------------
  【闲聊处理】
  - 快速匹配：你好/hi、你是谁、你能做什么、谢谢、再见 等 30+ 模式
  - AI 兜底：未匹配时调 DeepSeek 自然回复

  【上下文理解】
  - "这个任务删除" → 结合上一条AI回复解析"这个"
  - "第一个"/"第二个" → 从上一条回复中提取对应任务

  【防重复提交】
  - 发送时按钮变灰"处理中..."
  - 加载动画"思考中..."动态省略号

  【纯文本输出】
  - 强制禁止 Markdown 格式符号
  - 任务列表每一项独占一行，用 1. 2. 3. 编号

  【快捷按钮】
  - 📋 未完成 / ✅ 已完成 / 📅 今日任务
  - 🎤 语音按钮（已禁用，预留）


八、数据库结构
--------------------------------------------------------------------------------
  数据库: PostgreSQL (Supabase)

  【teams 表】
    字段       类型      说明
    id         INTEGER   主键自增
    name       TEXT      小组名
    admin_name TEXT      管理员姓名

  【users 表】
    字段                类型      说明
    id                  INTEGER   主键自增
    name                TEXT      用户名
    is_admin            BOOLEAN   是否管理员
    can_create          BOOLEAN   可布置任务
    can_remind          BOOLEAN   可催促他人
    password_hash       TEXT      密码哈希
    security_question   TEXT      密保问题
    security_answer_hash TEXT    密保答案哈希
    team_id             INTEGER   所属小组ID

  【records 表】
    字段        类型      说明
    id          INTEGER   主键自增
    name        TEXT      负责人
    action      TEXT      任务描述
    status      TEXT      未完成/已完成/进行中
    note        TEXT      原始输入文本
    deadline    TEXT      截止时间 YYYY-MM-DD HH:MM（可为NULL）
    created_at  TEXT      创建时间
    is_public   BOOLEAN   是否公共任务
    assigned_to TEXT      指定人员列表(逗号分隔)
    user_id     TEXT      创建者ID
    team_id     INTEGER   所属小组ID（强制数据隔离）

  【notifications 表】
    字段        类型      说明
    id          INTEGER   主键自增
    task_id     INTEGER   关联任务ID
    target_user TEXT      通知目标用户
    message     TEXT      通知内容
    is_read     BOOLEAN   是否已读
    created_at  TEXT      创建时间
