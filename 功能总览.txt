一、核心架构
--------------------------------------------------------------------------------
  app.py          Flask 后端，路由 + AI 调度
  db_utils.py     数据库抽象层（SQLite → 未来可换 Supabase）
  index.html      单页聊天界面

  API 路由：/ /chat /add /query /tasks /voice


二、AI 意图识别（classify_and_extract）
--------------------------------------------------------------------------------
  一次 AI 调用同时完成意图判断 + 数据提取，支持 6 种意图：

  add       录入    "张三有一个XX任务，后天要交"
                     提取 name/action/status/deadline → 入库
  query     查询    "今天有哪些任务"、"XX在哪天"
                     生成SQL → 执行 → 口语化 + 模糊匹配
  delete    删除    "清空今天的任务"、"把这个删掉"
                     预览 → 反问确认 → 确认后执行
  update    更新    "把XX改成已完成"、"XX改到后天"
                     更新 status 和/或 deadline
  confirm   确认    "确定"、"是的"
                     执行暂存的 pending 操作
  cancel    取消    "算了"、"取消"
                     丢弃暂存操作


三、时间理解能力
--------------------------------------------------------------------------------
  今晚十点前          当天 22:00
  今天要交/今天完成    当天 23:59
  明天下午3点         明天 15:00
  后天/后天上午       +2天
  大后天              +3天
  本周五前            本周五 18:00
  下周一              下周一 09:00
  7月份工作开题报告    "7月份"=主题描述，不误判为 deadline

  温度=0，同一输入永远返回相同结果


四、对话能力
--------------------------------------------------------------------------------
  上下文理解          "把这个任务删除" → 结合上一条AI回复解析"这个"指什么
  删除二次确认        先预览反问 → 用户"确定"后才真删 → "算了"则取消
  模糊匹配            查询"提交实验报告在哪天" → 数据库有近似任务时主动提示
  超时检测            查询时检测 deadline < now 且未完成 → 标记"已超时"
  纯文本输出          禁止 Markdown 符号（** * # - 等）


五、前端界面
--------------------------------------------------------------------------------
  消息时间戳          每条消息下方显示 14:30
  多行输入框          textarea 自动增高（最大120px），Enter发送
  快捷按钮            📋 未完成 / ✅ 已完成 / 📅 今天
  语音按钮占位        🎤 语音（已禁用，注释说明 Web Speech API 接入方式）
  加载动画            "思考中..." 动态省略号
  防重复提交          发送时按钮变灰"处理中..."
  任务列表弹窗        右上角📋按钮 → 表格展示所有任务（绿色已完成/橙色未完成/红色超时 + 统计）
  超时警告            查询结果含超时任务时，追加红色警告条


六、数据库结构
--------------------------------------------------------------------------------
  表名: records

  字段          类型      说明
  id           INTEGER   主键自增
  name         TEXT      负责人
  action       TEXT      任务描述
  status       TEXT      未完成/已完成/进行中
  deadline     TEXT      截止时间 YYYY-MM-DD HH:MM（可为NULL）
  created_at   TEXT      创建时间
  user_id      TEXT      预留（NULL），未来多用户支持
